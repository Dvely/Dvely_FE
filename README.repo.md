# AI 웹서비스 수정·배포 에이전트

비개발자와 입문개발자가 **자연어 대화만으로** 웹서비스를 만들고, 고치고, 미리보기로 확인하고, 배포까지 이어갈 수 있도록 돕는 서비스입니다.

---

## 왜 이 프로젝트를 만드나요?

생성형 AI 덕분에 웹페이지 초안은 빨리 만들 수 있게 됐지만, 실제 서비스 운영 단계에서 많은 사용자가 멈춥니다.

- 배포 지식 부족으로 런칭 실패
- 외주 수정 반복으로 비용 누적
- 코드 이해 장벽으로 작은 수정도 어려움
- 도메인/HTTPS/운영 설정이 심리적 허들로 작동

이 프로젝트는 그 간극을 줄여, **“만들기”에서 “운영 가능한 상태”로 넘어가는 마지막 1km**를 해결하는 것이 목표입니다.

---

## 한 줄 가치 제안

**대화만으로 제작/수정/검증/배포 준비까지.**  
복잡한 배포 지식 없이도 실제 공개 가능한 URL까지 도달하게 합니다.

---

## 누구를 위한 서비스인가요?

- 비개발자 (창업자, 1인 사업자, 소상공인)
- 입문개발자 (프론트 기초는 있지만 배포/운영 경험이 부족한 사용자)
- 외주 산출물을 직접 유지보수하고 싶은 고객
- 포트폴리오를 직접 운영하고 싶은 취준생/프리랜서

---

## 핵심 사용 흐름

1. 프로젝트 시작 (새로 만들기 / ZIP 업로드 / GitHub 불러오기)
2. AI Agent에 자연어 요청
3. 코드 수정 및 빌드
4. 내부 미리보기에서 실제 동작 확인
5. 승인 후 반영
6. 배포 승인 및 자동화 파이프라인 실행
7. 배포 URL 제공, 필요 시 도메인 연결

> 기본 실행 순서: **변경사항 반영 -> 빌드/미리보기 -> 배포 -> 도메인 연결**

---

## MVP 핵심 기능

### 1) 프로젝트 진입

- 프로젝트 목록 화면
- 새 프로젝트 생성
- ZIP 업로드
- GitHub 저장소 불러오기

### 2) 대화형 개발

- 자연어로 UI/기능/오류 수정 요청
- 기존 대화 이어서 작업
- 올인원 요청 지원 (수정 + 배포 + 도메인 연결 등)

### 3) 미리보기와 정확성

- 실제 빌드 기반 내부 미리보기 제공
- 클릭/입력 등 테스트 가능한 실행 환경
- 빌드 실패 시 쉬운 설명 + 우선 해결안 제시

### 4) 승인 및 반영 정책

- 승인 대상: 변경 반영 / 배포 / 도메인 연결
- 자동 승인 세분화: 전체, 수정, 배포, 도메인 개별 설정
- 요청 단위별 작업 이력 추적

### 5) GitHub 및 배포

- GitHub를 기본 이력/백업 채널로 사용
- preview 브랜치 기반 검증
- 승인된 변경을 기준으로 배포 워크플로우 실행
- MVP 기본 배포 대상: 정적 사이트 (GitHub Pages)

- "최신 승인 버전으로 배포할까요?" 안내
- 배포 버전 / 레포 / URL / 변경 요약 표시

### 화면 5. 설정
- GitHub 연동 상태
- 자동 승인 정책
- 배포/도메인 기본 정책

## 비기능 목표 (MVP)

- **신뢰성**: 배포 단계 성공률 극대화
- **정확성**: 미리보기와 실제 배포 결과 일치성 강화
- **사용성**: 비개발자도 이해 가능한 UX
- **응답성**: 요청 후 미리보기 반영 시간 최소화
- **추적성**: 요청 -> 코드 변경 -> 배포 결과 연결 가능
- **확장성**: 장기적으로 AWS/GCP 연동 오케스트레이션으로 확장

## 프로젝트 방향

이 제품은 자체 호스팅 플랫폼을 직접 구축하기보다,  
**AI 코드 작업 + 외부 배포 인프라 자동화 연결**을 제공하는 오케스트레이션 레이어를 지향합니다.

즉, 사용자는 기술 디테일보다 결과에 집중하고, 시스템은 안전한 승인 흐름으로 반영/배포를 돕습니다.
